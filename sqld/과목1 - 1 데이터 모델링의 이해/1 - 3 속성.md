> 속성: 업무에서 필요한 인스턴스를 관리하고자 하는 의미상 분리되지 않는 최소의 데이터 단위

- 한 개의 엔터티는 2개 이상의 인스턴스 집합
- 한 개의 엔터티는 2개 이상의 속성을 가짐
- 한 개의 속성은 1개의 속성값을 가짐

> 구성 방식의 분류

- PK
- FK
- 일반 속성

> 속성의 분류

- 기본: 업무로부터 추출한 모든 일반적인 속성
- 설계: 업무를 규칙화하기 위해 새로 만들거나 변형, 정의하는 속성 ex) 일련번호
- 파생: 다른 속성에 영향을 받아 발생하는 속성. 빠른 성능을 낼 수 있도록 원래 속성의 값을 계산, 적을 수록 좋음. ex) 합

> 도메인: 속성에 대한 데이터 타입, 크기, 제약사항 지정

> 속성의 명명

1. 해당업무에서 사용하는 이름 부여
2. 서술식 속성명은 사용 금지
3. 약어 금지
4. 구체적으로 명명하여 데이터 모델에서 유일성 확보(직원.이름과 고객.이름를 같게 하지 말 것!)
