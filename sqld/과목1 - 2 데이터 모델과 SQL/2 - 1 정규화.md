> 정규화

- 데이터가 종속된 테이블에 적절하게 배치되도록 하는 것
- 논리 데이터 모델 상세화 과정의 대표적인 활동으로, 정규화는 개념 데이터 모델의 일관성을 확보하고 중복을 제거하여 속성들이 가장 적절한 엔터티에 배치되도록 한다.
- 제1정규형
  - 모든 속성은 반드시 하나의 값을 가져야 한다.
  - 같은 성격, 내용 컬럼이 연속될 때 컬럼제거, 테이블 생성
- 제2정규형
  - 엔터티의 일반속성은 주식별자 전체에 종속이어야 한다.
  - PK 복합키 구성일 대 부분적 함수 종속 관계 테이블 분리
- 제3정규형
  - 엔터티의 일반속성 간에는 서로 종속적이지 않다.
  - PK가 아닌 일반 컬럼에 의존하는 컬럼 분리

> 반정규화

- 정규화된 엔터티, 속성, 관계에 대해 시스템 성능 향상과 개발과 운영의 단후롸를 위해 중복, 통합, 분리 등을 수행하는 데이터 모델링 기법
- 조회 시 디스크의 I/O가 많거나 경로가 멀어 조인에 의한 성능 저하를 막기 위해 수행
- 일반적으로 정규화 시 입력/수정/삭제 성능이 향상되며 반정규화 시 조인의 성능이 향상된다.
